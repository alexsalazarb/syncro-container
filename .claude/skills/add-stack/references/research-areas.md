# Research Areas

Used by [add-stack](../SKILL.md) Step 3 to research a new stack before writing templates.

---

## Required research topics

For each new stack, research these areas to produce accurate templates and standards:

| Topic | Search queries | Used in |
|-------|---------------|---------|
| Project structure | "{stack} recommended project structure 2024" | AGENTS template: Patterns section |
| Naming conventions | "{stack} file naming conventions" | AGENTS template: Naming Conventions |
| CLI commands | "{stack} CLI commands cheatsheet" | AGENTS template: Quick Reference |
| Best practices | "{stack} best practices {year}" | Standards doc |
| Common anti-patterns | "{stack} common mistakes anti-patterns" | AGENTS template: Things to Avoid |
| State management | "{stack} state management approaches" | AGENTS template: Critical Constraints |
| Testing | "{stack} testing best practices" | Standards doc: Testing section |
| Accessibility | "{stack} accessibility guidelines" | AGENTS template: Critical Constraints |
| Security | "{stack} security best practices OWASP" | Standards doc: Security section |
| Detection file | "{stack} project config file" | init-repo.sh: detect_stack() |

## Minimum research

At least **3 web searches** covering:
1. Official documentation / style guide
2. Community best practices (recent articles, conference talks)
3. Common pitfalls and anti-patterns

## Quality criteria

- Prefer official documentation over blog posts
- Prefer recent sources (within 2 years)
- Cross-reference multiple sources for contested practices
- When practices conflict between sources, note the mainstream default and
  mention the alternative in a comment
